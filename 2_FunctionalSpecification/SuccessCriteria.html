Success Criteria

If this project needs to be successful the specification for the system should do what it is required to do. By considering the user needs the success of the system can be determined.

  • Controlling the availability and preventing overbooking of rooms.

    1. Checking on available and unavailable room will not take more than 20 seconds.

    2. Setting up a room to available or unavailable will not take more than 20 seconds.

  • Making the users work run smoother and well organized.

    1. Entering details will not be handwritten. Every detail will be typed using a computer.

    2. Collecting sheets of papers and trying to organise them will not be necessary, because every details will be organised by the database itself.

  • Guarantee customer satisfaction.

    1. It will not take more than 5 minutes to make a reservation for the guest. It depends on how fast the guest provides his/hers details to the user or how fast the user types.

  • Enable the user to add new or edit existing Guest Detail.

    1. Adding a guest detail will not take more than 1 minute.

    2. A hyperlink is placed on the GuestID, so editing a guest details will not be difficult to do.

  • Enable the user to add new or edit existing Room Detail.

    1. Adding a new room details will not take more than 1 minute.

    2. If a new room details is been entered, at the same time the user can state whether that room is available or not.

    3. A hyperlink is placed on the RoomID, so editing a room details will not be difficult to do.

    4. Finding a room detail will not involve the user to type anything.

  • Enable the user to add new or edit existing Booking Detail.

    1. Adding a new booking detail will not take more than 2 minutes.

    2. A hyperlink is placed on the BookID, so editing a booking details will not be difficult to do.

    3. Finding a booking detail will not involve the user to type anything.

  • Enable the user to add new or edit existing Extra Charge Detail.

    1. Adding a new Extra Charge detail will not take more than 1 minute.

    2. A hyperlink is placed on the ExtraChargeID, so editing an extra charge details will not be difficult to do.

    3. Finding an extra charge detail will not involve the user to type anything.

  • Enable the user to add new or edit existing Payment Detail. 

    1. Adding a new payment detail will not take more than 1 minute.

    2. A hyperlink is placed on the PaymentID, so editing a payment details will not be difficult to do.

    3. Finding a payment detail will not involve the user to type anything.

  • Enable the user to delete a booking. 

    1. Deleting a booking detail will not take more than 30 second.

    2. Deleting a booking detail will be done automatically.

    3. A booking details can’t be deleted accidently.

    4. Finding a booking cancelation detail will not involve the user to type anything.

  • Easy to navigate through Guest details, Room details, Booking details, Payment details, ExtraCharge details, Booking cancelation details and Reports. Every details are available in the Main Menu form.

    1. Navigating through all the forms will not take more than 20 seconds.

    2. Editing guest details and room details can be done in the Booking details form. Hyperlinks are placed on the GuestID and RoomID. This method is useful, because the user doesn't need to go to Guest details form or Room details form to edit the details. Guest details and room details can be edited while staying in the Booking details form.  

  • Enabling the user to view and print of reports.

    1.  Printing and view a report will not take more than 30 seconds.

  • Generating monthly and annual income reports to see the outcome of the organisation.

    1. Calculation should be 100% correct and done automatically.

  • Creating a search that enables the user to find a particular guest.

    1. This can be done within 20 seconds. This will be more useful for the user because there will be 100 of guest details in the form. So finding one guest won’t be hard.

  • Grouping guests using their age, such as teens, adults and seniors.

    1. The age grouping will be used for generating reports related to the age group. This will be useful for the organisation to know in what age group has made more bookings.

  • To provide accessibility control to data with respect to users.

    1. Two user levels are introduced to the system, who are manager/organiser and staff. the manage/organiser will have full authority of the system, where the staff will have limited access.

  • The ability to make booking anytime.

    1. The system is 24h available for the user to make bookings for the guest.

  • To avoid manual and repetitive work.

    1. Everything will be done automatically other than typing and navigation through forms.

    2. Using the system will be a new and exciting thing for the user. They will also learn how the new technology works.

  • Creating a User Guide that will help the user to know how to install the system, use the system and how to troubleshoot problems. 

    1. If the user gets into trouble with the system or did something wrong, he/she can check the user guide and troubleshoot the problem.

  • Disabling the Ribbon menu, navigation pane and the Shortcut Menu.

    1. To make the database look more professional.

    2. By disabling these features, it will help the user be more focused with work without getting distracted or interfered.
